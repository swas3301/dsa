Day 4 of my DSA Journey 🚀
Solved: Implement Stack using Queues (LeetCode #225)

Today’s problem flipped the usual perspective—implementing a LIFO stack using FIFO queues. It was a great exercise in understanding how data structures can be adapted creatively.

🔹 Key Learning:
Even when constraints seem restrictive, we can simulate required behavior:

* Use queue operations to mimic stack behavior
* Reorder elements so the most recent stays at the front

🔹 Approach:
Used a single queue and rotated elements after each push so that the last inserted element always comes to the front (acting as the stack top).

🔹 Complexity:
Push: O(n)
Pop: O(1)
Top: O(1)
Space: O(n)

💡 This problem strengthened my understanding of how stacks and queues differ, and how to bridge that gap with clever manipulation.
