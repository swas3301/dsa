Solved: Linked List Cycle (LeetCode #141)

Today’s problem was about detecting cycles in a linked list—a classic problem that tests traversal logic and data structure choice.

🔹 Key Learning:
To detect a cycle, we need to know if we revisit a node:

* If a node appears again, a cycle exists
* Hashing makes this check straightforward

🔹 Approach:
Used a set-based approach:

* Traverse the linked list node by node
* Store each visited node in a set
* If a node is already in the set → cycle detected

🔹 Complexity:
Time: O(n)
Space: O(n)

💡 This problem reinforced how hash-based lookups can simplify detection problems. Next step: solving it using Floyd’s Cycle Detection (fast & slow pointers) for O(1) space.
