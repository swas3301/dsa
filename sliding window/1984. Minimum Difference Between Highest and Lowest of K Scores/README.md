Day 18 of my DSA Journey 🚀
Solved: Minimum Difference Between Highest and Lowest of K Scores (LeetCode #1984)

Today’s problem was a clean sliding window problem that showed how sorting can simplify comparisons.

🔹 Key Learning:
To minimize the difference between highest and lowest scores:

* Sort the array first
* Check only groups of k consecutive elements
* The minimum difference will be within one of these windows

🔹 Approach:
Used sorting + sliding window:

* Sort the scores in ascending order
* Use two pointers to maintain a window of size k
* Track the minimum difference between the first and last element in each window

🔹 Complexity:
Time: O(n log n)
Space: O(1) *(excluding sorting space)*

💡 This problem reinforced how sorting can transform a seemingly complex selection problem into a simple window-based solution.
